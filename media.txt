# Django Static/Media Files Setup Guide
# For: Christian Fundraising Foundation
# Best Practices for Single-Admin Sites

## RECOMMENDED STRUCTURE
project_root/
├── static/       # CSS/JS/fonts
├── assets/       # Website images (treated as static)
└── media/        # Future uploads (keep empty)

## ACTION STEPS

1. CREATE FOLDERS
   mkdir -p static/assets/images
   mkdir media

2. MOVE EXISTING IMAGES
   mv media/*.jpg static/assets/images/
   mv media/*.png static/assets/images/

3. UPDATE TEMPLATES
   Replace all:
   src="{% static 'media/image.jpg' %}"
   With:
   src="{% static 'assets/images/image.jpg' %}"

4. CONFIGURE SETTINGS (base.py)
   STATICFILES_DIRS = [
       BASE_DIR / "static",  # Custom static files
   ]
   STATIC_ROOT = BASE_DIR / "staticfiles"  # For collectstatic
   MEDIA_ROOT = BASE_DIR / "media"         # Future-proof
   MEDIA_URL = '/media/'

5. RUN COLLECTSTATIC
   python manage.py collectstatic --noinput

6. VERIFY (run these commands)
   # Check image paths:
   python manage.py findstatic assets/images/home-image.jpg

   # Test URL resolution:
   python manage.py shell
   >>> from django.templatetags.static import static
   >>> print(static('assets/images/home-image.jpg'))

## WHY THIS WORKS
- Keeps designed assets (images/CSS/JS) separate from potential uploads
- Static files get proper caching during deployment
- Zero downtime migration
- Room to grow if needs change

## ALTERNATIVE (All-in-media)
Only use if absolutely necessary:

1. settings.py:
   STATICFILES_DIRS = [BASE_DIR / "media"]
   
2. Templates keep:
   src="{% static 'media/image.jpg' %}"
   
3. Run:
   python manage.py collectstatic

⚠️ Not recommended long-term
